# 02. Схема базы данных SQLite

> Статус: ⬜ Не начато

## Цель

Спроектировать схему SQLite для системы памяти: 4 слоя, общая/личная память агентов, FTS5-индексы для полнотекстового поиска, sqlite-vec для векторного поиска.

## 4 Слоя памяти

| Слой         | Назначение                         | Формат             | Жизненный цикл                        |
| :----------- | :--------------------------------- | :----------------- | :------------------------------------ |
| 1. Фокус     | Системный промт, самоидентификация | Markdown           | Вручную / редко меняется              |
| 2. Контекст  | Активные проекты, текущие задачи   | Markdown + YAML FM | Обновляется по релевантности          |
| 3. Архив     | Сжатые выводы, паттерны, инсайты   | Markdown (EN)      | Результат ночного цикла               |
| 4. Сырой лог | Полная история запросов/ответов    | Plain Text (RU)    | Append-only, очищается при компрессии |

## Traceability: request_id как сквозной ключ

Каждый запрос получает уникальный `request_id` (UUID). Он прошивается через все слои:

```
Слой 4 (сырой лог)     → request_id = "abc-123"
Слой 3 (архив)         → source_request_ids = ["abc-123", "def-456"]
Слой 2 (контекст)      → derived_from = ["слой-3-id-789"]
```

Это позволяет:

- **Ночной верификации** — развернуть цепочку: Слой 3 факт → Слой 4 сырой лог → проверить точность компрессии
- **Пользовательской ревизии** — найти откуда взялся конкретный «факт» в памяти
- **Откату** — если факт в Слое 3 оказался галлюцинацией, найти и пересжать из оригинала

## Таблицы (предварительно)

```sql
-- Слой 1: Фокус (key-value для identity/directives)
-- Слой 2: Контекст (documents с YAML frontmatter, derived_from[])
-- Слой 3: Архив (compressed knowledge, source_request_ids[])
-- Слой 4: Сырой лог (append-only, request_id, session_id)
-- Общая память (факты о пользователе)
-- Личная память агентов (per-role experience)
-- FTS5 виртуальные таблицы
-- sqlite-vec таблица для embeddings
```

## Индексы

- **FTS5:** По полям content, tags для Слоёв 2 и 3
- **sqlite-vec:** Embeddings от `nemoretriever-300m` (размерность — уточнить)
- **B-tree:** По agent_id, created_at, layer
