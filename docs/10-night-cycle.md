# 10. Ночной цикл: PII → Translate → Compress → Deduplicate

> Статус: ⬜ Не начато

## Цель

Автоматизированный cron-процесс для компрессии Слоя 4 → Слой 3 с очисткой от PII, переводом на EN и дедупликацией знаний.

## Pipeline

```
Слой 4 (сырые логи, RU, plain text)
        │
        ▼
┌─────────────────────┐
│  1. PII Detection   │  ← nvidia/gliner-pii
│  Удалить: имена,    │
│  адреса, телефоны,  │
│  email, карты       │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│  2. Translation     │  ← nvidia/riva-translate-4b-instruct-v1_1
│  RU → EN            │
│  (повышение Token   │
│   Density)          │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│  3. Compression     │  ← stepfun-ai/step-3.5-flash
│  Plain Text → MD    │
│  Извлечь: решения,  │
│  инсайты, паттерны  │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│  4. Deduplication   │  ← step-3.5-flash + vector similarity
│  Сравнить с Слой 3  │
│  Merge / Update /   │
│  Append             │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│  5. Anti-patterns   │  ← step-3.5-flash
│  "На чём застряли"  │
│  → отдельная запись  │
│  в Слой 3           │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│  6. Resolve         │  ← step-3.5-flash
│  Противоречия с     │
│  существующими      │
│  знаниями           │
└─────────────────────┘
```

## Расписание

- **Триггер:** Cron, ежедневно в 03:00 (или при неактивности > 4 часов)
- **RPM:** Ночью весь бюджет 40 RPM свободен

## RPM Budget (одна ночная сессия)

| Этап               | Вызовов   | RPM   |
| :----------------- | :-------- | :---- |
| PII detection      | N записей | ~N    |
| Translation        | N записей | ~N    |
| Compression        | Batch     | ~5-10 |
| Dedup + similarity | Batch     | ~5-10 |
| Anti-patterns      | 1         | 1     |
| Resolve            | 1-5       | 1-5   |

При 40 RPM можно обработать ~20 записей за минуту, ~1200 за час.

## Верификация при компрессии

Каждая запись в Слое 3 хранит `source_request_ids[]` — ссылки на оригинальные записи Слоя 4.

Ночной цикл добавляет этап **Verify**:

```
3. Compression → Слой 3 entry (с source_request_ids)
       │
       ▼
3.5 Verify (step-3.5-flash):
    - Взять source записи из Слоя 4
    - Сверить сжатый факт с оригиналом
    - Если расхождение → пометить confidence: LOW
    - Если ок → confidence: HIGH
```

Записи с `confidence: LOW` попадают в очередь на ручную ревизию пользователем (будущий UI).

## Ручная ревизия памяти (TODO: будущий UI)

- Пользователь может просмотреть Слой 3, кликнуть на запись → развернуть до Слоя 4 (через `source_request_ids`)
- Отредактировать / удалить / подтвердить факт
- Подтверждённые факты получают `verified: true`, повышенный приоритет в RAG
