# 07. Auth Middleware

> Статус: ✅ Реализовано

## Цель

Защита прокси-сервера от несанкционированного доступа. Любой процесс на машине без токена не должен иметь доступа к API.

## Механизм

- **Bearer-токен** в заголовке `Authorization: Bearer <token>`
- Токен задаётся в env-переменной `PROXY_AUTH_TOKEN`
- Валидация на уровне Elysia middleware (до роутов)

## Поведение

| Сценарий                    | HTTP Status | Тело ответа                                 |
| :-------------------------- | :---------- | :------------------------------------------ |
| Нет заголовка Authorization | 401         | `{"error": "Missing authorization header"}` |
| Невалидный токен            | 401         | `{"error": "Invalid token"}`                |
| Валидный токен              | → next()    | —                                           |

## Исключения

- `GET /health` — без авторизации (для мониторинга)
