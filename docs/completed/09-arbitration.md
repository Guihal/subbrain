# 09. Протокол арбитража «Общей комнаты»

> Статус: ⬜ Не начато

## Цель

Определить как работает режим «Общая комната»: параллельный вызов нескольких специалистов и синтез финального ответа Тимлидом. Ключевое преимущество — бесплатный доступ к NVIDIA NIM, что делает мульти-агентный подход экономически бесплатным (ограничение только RPM).

---

## Когда активируется

### Явные триггеры (от пользователя)

- «обсудите», «покажите разные подходы», «что думает команда»
- Указание конкретных ролей: «спроси кодера и критика»

### Автоматические триггеры (определяет Flash-агент на pre-processing)

Flash-агент (`nemotron-mini-4b`) классифицирует входящий запрос:

| Сигнал                | Пример                                         | Действие                         |
| :-------------------- | :--------------------------------------------- | :------------------------------- |
| Архитектурное решение | «как организовать...», «какой подход лучше...» | → Общая комната                  |
| Выбор из альтернатив  | «X или Y?», «плюсы и минусы»                   | → Общая комната                  |
| Код-ревью             | «проверь этот код», diff > 100 строк           | → Общая комната (coder + critic) |
| Простое кодирование   | «напиши функцию...», «исправь баг»             | → Single-model (coder)           |
| Простой вопрос        | «чем отличается X от Y?»                       | → Single-model (generalist)      |
| Рутина                | «переименуй», «добавь импорт»                  | → Single-model (flash)           |

Порог: если Flash определяет `complexity_score > 0.7` → Общая комната. Иначе → single-model.

## Early Exit (быстрый путь)

Не всегда нужно ждать всех. Стратегия **first-confident**:

```
Promise.all() запущен, но:

1. Первый агент ответил за 3 сек, confidence: HIGH
   → Таймер: ждём ещё 5 сек для остальных
   → Если остальные не успели → Тимлид синтезирует из того, что есть

2. Первый ответил, confidence: LOW
   → Ждём всех (до timeout 30 сек)

3. Timeout 30 сек
   → Синтез из того, что пришло (даже если 1 из 3)
```

**Определение confidence:** Flash-агент анализирует ответ (длина, наличие «не уверен», количество альтернатив). Быстрая эвристика, без дополнительного RPM.

## Промт Тимлида для синтеза

```markdown
## Твоя роль

Ты Тимлид. Тебе дали один запрос и три ответа от специалистов.
Синтезируй единый ответ.

## Ответы специалистов

### Кодлер (devstral-123b) — вес: {coder_weight}

{coder_response}

### Критик (kimi-k2-thinking) — вес: {critic_weight}

{critic_response}

### Генералист (qwen3-coder-480b) — вес: {generalist_weight}

{generalist_response}

## Инструкции

1. Найди **консенсус** — что все согласны.
2. Отметь **разногласия** — кто с кем не согласен и почему.
3. Прими **решение** с обоснованием.
4. Если разногласие критическое — покажи пользователю оба варианта.
5. Формат ответа: как если бы ты один отвечал (не упоминай агентов).
```

## Weighted Voting (динамические веса)

Веса агентов **не фиксированные** — они обновляются на основе истории.

### Начальные веса

| Агент              | Код     | Архитектура | Ревью   | Reasoning |
| :----------------- | :------ | :---------- | :------ | :-------- |
| Coder (devstral)   | **1.5** | 0.8         | 1.0     | 0.7       |
| Critic (kimi-k2)   | 0.8     | 1.0         | **1.5** | **1.5**   |
| Generalist (qwen3) | 1.0     | **1.3**     | 1.0     | 1.0       |

### Обновление весов

Post-processing записывает в Слой 3:

```markdown
## Agent Performance Log

- 2026-04-17: task=architecture, winner=critic
  coder предложил X, но critic нашёл race condition → critic_weight[architecture] += 0.1
```

Формула обновления (EMA — Exponential Moving Average):

```
new_weight = old_weight * 0.9 + outcome * 0.1
```

Где `outcome = 1.0` если ответ агента вошёл в финальный, `0.0` если отвергнут.

## RPM Budget

```
1 запрос «Общая комната» (полный цикл):
  Pre-processing (flash + embed + rerank):  3 RPM
  Dispatch (тимлид):                        1 RPM
  3 специалиста (параллельно):              3 RPM
  Synthesis (тимлид):                       1 RPM
  Post-processing (flash + embed):          2 RPM
  ─────────────────────────────────────────
  Итого:                                   10 RPM
  → 4 полных запроса в минуту

Оптимизированный (early exit, 2 агента):
  Pre-processing:                           3 RPM
  2 специалиста:                            2 RPM
  Synthesis (тимлид):                       1 RPM
  Post-processing:                          2 RPM
  ─────────────────────────────────────────
  Итого:                                    8 RPM
  → 5 запросов в минуту
```

## Разрешение конфликтов

| Сценарий             | Стратегия                                                 | Пример                                                              |
| :------------------- | :-------------------------------------------------------- | :------------------------------------------------------------------ |
| Все согласны         | Тимлид объединяет лучшее из каждого                       | Все предлагают Strategy pattern → merge деталей                     |
| 2 из 3 согласны      | Majority wins, dissent логируется                         | Coder и Generalist за подход A, Critic против → A, но отмечаем risk |
| Все расходятся       | Тимлид выбирает на основе весов + презентует альтернативы | Weighted vote → рекомендация, но «также рассмотри...»               |
| Критический конфликт | Эскалация к пользователю                                  | Безопасность vs производительность → явный вопрос                   |

### Определение «критического конфликта»

Конфликт критический, если:

- Касается безопасности (XSS, SQL injection, auth)
- Касается необратимых действий (миграция БД, удаление данных)
- Разница в оценке сложности > 3x между агентами

## Конфигурация участников комнаты

Не каждая задача требует всех трёх. Flash определяет набор:

| Тип задачи  | Участники                   | RPM |
| :---------- | :-------------------------- | :-- |
| Архитектура | Coder + Critic + Generalist | 10  |
| Код-ревью   | Coder + Critic              | 8   |
| Выбор стека | Generalist + Critic         | 8   |
| Сложный баг | Coder + Generalist          | 8   |

---

## Передача контекста

Все агенты получают **одинаковый** Executive Summary от pre-processing (не дублируем RAG). Разница — только в system prompt:

```
System prompt = Identity(role) + Executive Summary + MCP Tools
```

Специалисты **могут** использовать MCP tools (memory_search) внутри своего вызова, если модель поддерживает function calling. Это не тратит дополнительный RPM на NVIDIA (tools обращаются к локальной SQLite).
