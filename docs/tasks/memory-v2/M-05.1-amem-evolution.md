# M-05.1 · A-MEM neighbour evolution (tag merge on insert)

**Tier:** P2 · **Effort:** S · **Deps:** M-05 (memory_edges) — landed · **Status:** DONE
**Migration assignment:** **none** (extends `tags` column, no schema change).

## Цель

A-MEM (Agentic Memory, Wang et al. 2024 — Zettelkasten-style auto-organising memory) describes attribute evolution: when a new note is linked to a neighbour, the neighbour absorbs novel attributes from the new note. M-05 положил `linkRelated` post-hook, который рисует `relates` edges на top-3 vec neighbours, но neighbours остаются immutable — их `tags` не обновляются от нового контекста.

После M-05.1: при `linkRelated` для каждого drawn edge → `evolveNeighbour(memory, neighbourId, neighbourLayer, insertedTags)`:
1. Прочитать current tags neighbour'а (`getContext / getShared`).
2. Распарсить `tags` (CSV строка; пустая = []).
3. Слить с insertedTags (unique, preserve order, новые в tail).
4. Cap до `MAX_TAGS` (default 10) — keep newest (drop oldest from head).
5. Если diff с current → `updateContext / updateShared({tags})` (allow-list уже включает `tags`).

NO content change — только tags evolve. NO confidence / status / kind касаемо. NO LLM call (pure-string merge).

Foundation для **M-05.2** (LLM contradiction detection) — uses evolved tag-corpus to seed prompts, и **M-09** cross-layer dedup которая JOIN'ит по category + tag overlap.

## Изменение

### `evolveNeighbour` shape

```ts
const MAX_TAGS_DEFAULT = 10;

function evolveNeighbour(
  memory: MemoryDB,
  neighbourId: string,
  neighbourLayer: "context" | "shared",
  insertedTags: string[],
  maxTags: number,
): void {
  if (insertedTags.length === 0) return;

  const row =
    neighbourLayer === "context"
      ? memory.getContext(neighbourId)
      : memory.getShared(neighbourId);
  if (!row) return;

  const currentTags = parseTags(row.tags ?? "");
  const merged = mergeUnique(currentTags, insertedTags, maxTags);
  if (sameSet(currentTags, merged)) return; // no-op

  const newTagsCsv = merged.join(",");
  if (neighbourLayer === "context") {
    memory.updateContext(neighbourId, { tags: newTagsCsv });
  } else {
    memory.updateShared(neighbourId, { tags: newTagsCsv });
  }
}
```

Helpers (also module-private, ≤30 LOC всего):
- `parseTags(csv)` — split `,`, trim, drop empty.
- `mergeUnique(a, b, cap)` — preserve order; drop dups; tail-truncate to cap (keep newest).
- `sameSet(a, b)` — order-insensitive equality.

### Hook integration в `linkRelated`

Plan signature change: pass `insertedTags` argument so the function doesn't need a second DB read for the inserted row. Все callers `linkRelated` (writeShared / writeContext extractors в M-05) передают tags из insert payload — уже доступно.

Diff (concept):
```ts
export async function linkRelated(
  memory: MemoryDB,
  rag: RAGPipeline,
  insertedId: string,
  layer: "context" | "shared",
  content: string,
  insertedTags: string[],   // NEW
  log: RequestLogger,
): Promise<void> {
  // ... existing rag.search + edge draw loop ...
  for (const n of neighbours) {
    // ... existing skip-self + linkEdge ...
    if (insertedTags.length > 0 && evolveEnabled()) {
      try {
        evolveNeighbour(memory, n.id, n.layer, insertedTags, maxTags());
      } catch (err) {
        // Swallow — evolution is best-effort, never blocks the write.
        log.warn("post.extractors", `evolveNeighbour failed for ${n.id}: ${...}`);
      }
    }
  }
}
```

`evolveEnabled()` reads `process.env.LINK_EVOLVE_TAGS_ENABLED` (default `true`). `maxTags()` reads `LINK_EVOLVE_MAX_TAGS` (default 10). Both read at call-time per repo pattern.

### Caller updates

`packages/agent/src/pipeline/agent-pipeline/post/extractors.ts` — both `writeShared` and `writeContext` have `tags` arg; pass into `linkRelated(memory, rag, id, layer, content, tags, log)`. Trivial update (one extra arg per call site, ≤4 lines total).

### Existing API

`memory.getShared(id)` (через `MemoryDB` facade) уже есть — `packages/core/src/db/tables/shared.ts:159` + `packages/core/src/db/index.ts:208`. Reuse — no new SQL helper.

### Edge cases handled

- **insertedTags empty** — early return, no DB writes.
- **neighbour deleted between linkEdge и evolveNeighbour** — getContext/getSharedById returns null → early return.
- **merged === current** — no-op (sameSet check), avoid spurious updated_at bumps.
- **>MAX_TAGS already on neighbour** — merged includes new tags, then tail-truncated; oldest tags dropped.
- **evolveNeighbour throw** — caught + logged, doesn't abort linkRelated (same pattern as edge insert error).

### Env knobs

- `LINK_EVOLVE_TAGS_ENABLED` (default `true`).
- `LINK_EVOLVE_MAX_TAGS` (default 10).

## Тесты

`tests/post-link-related.test.ts` (extend) ИЛИ `tests/post-link-related-evolution.test.ts` (new):

1. **Empty inserted tags → no evolution** — `insertedTags=[]` → neighbours' tags unchanged.
2. **Pure-add merge** — current=`a,b`, inserted=`c,d` → result `a,b,c,d`.
3. **Dup deduplication** — current=`a,b`, inserted=`b,c` → result `a,b,c` (no double `b`).
4. **Cap eviction** — current=`a,b,c,d,e,f,g,h,i,j` (10), inserted=`new1,new2`, cap=10 → result `c,d,...,j,new1,new2` (drop a,b oldest).
5. **No-op when already covered** — current=`a,b,c`, inserted=`a,b` → no `updateContext` call (sameSet check).
6. **Layer routing** — context neighbour → `updateContext`, shared neighbour → `updateShared`.
7. **`LINK_EVOLVE_TAGS_ENABLED=false` → skip evolution**.
8. **Neighbour deleted mid-flight** — `linkEdge` succeeds, but neighbour row gone before evolve → no throw.

Test DB = `data/test-mem5.1-evolve.db` ИЛИ reuse existing test DB if extend.
