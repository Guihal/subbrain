# M-05.2 · LLM contradiction detection (auto kind='contradicts' edges)

**Tier:** P2 · **Effort:** S · **Deps:** M-05 (memory_edges + linkRelated) — landed; M-05.1 (tag evolution, evolved tag-corpus сидит prompt) — landed · **Status:** DONE
**Migration assignment:** **none** (extends `memory_edges`, schema unchanged — `kind='contradicts'` уже в CHECK constraint M-05).

## Цель

A-MEM (Wang et al. 2024) описывает `kind='contradicts'` как выделенный тип edge: новая memory отрицает / противоречит существующему соседу (e.g. "user prefers X" → "user no longer prefers X"). M-05 положил schema (CHECK включает `'contradicts'`), M-05.1 — `relates` + tag evolution. M-05.2 закрывает дыру: автоматически проставлять `contradicts` при insert, **только если LLM явно подтверждает противоречие** среди top-3 vec neighbours (тех самых, что уже стали `relates`).

После M-05.2:

- В `post/link-related.ts` ПОСЛЕ relates-loop (т.е. neighbours уже отобраны и tag-evolved) → один LLM call с inserted_content + список candidate-neighbours → JSON `{ "contradicts": [{"id": "...", "confidence": 0.0..1.0}] }`.
- Для каждого item с `confidence ≥ MIN_CONTRADICTION_CONF` (default 0.7) → `memory.linkEdge(insertedId, layer, n.id, n.layer, "contradicts", confidence)`.
- LLM call — **best-effort**, default OFF (cost), env-gated.
- Foundation для **M-06.5** (UI / curation tool: surface contradiction edges as conflict-list для user review) и **M-09** (cross-layer dedup использует contradiction-mass для merge priority).

NO content mutation. NO status flip (`active`→`stale` — это отдельная супер-задача supersedes-flip, out of scope). NO removal of `relates` edge — оба kind могут coexist (соседство И противоречие).

## Изменение

### LLM call shape

Single non-streaming `router.chat(MODEL, messages, "low")` call. Default `MODEL` через env `CONTRADICT_MODEL_ROLE` (default `"memory"` — gpt-5.1+MiniMax fallback, тот же что hippocampus / night-cycle).

Prompt (~150 tokens up):

```ts
const sys = "You detect direct contradictions between memory snippets. " +
            "Reply ONLY in JSON: {\"contradicts\":[{\"id\":\"<id>\",\"confidence\":0.0-1.0}]}. " +
            "Empty array if no contradiction. Do NOT explain. " +
            "A contradiction means the new fact directly negates a candidate (opposite preference, " +
            "reversed decision, conflicting attribute). Loose-relatedness ≠ contradiction.";
const user = `NEW: ${insertedContent.slice(0, 800)}\n\nCANDIDATES:\n` +
             candidates.map(c => `- id=${c.id}: ${c.content.slice(0, 400)}`).join("\n");
```

Hard caps: `insertedContent` 800 chars, each candidate 400 chars (sufficient for "user likes X" vs "user dislikes X"-style contrasts; we are NOT solving free-form NLI).

Parse: find first `{`, last `}`, `JSON.parse` substring. Bad JSON → log.warn + return `[]` (no edges drawn). One retry NOT performed (best-effort).

### Hook integration в `linkRelated`

```ts
export async function linkRelated(
  memory: MemoryDB,
  rag: RAGPipeline,
  router: ModelRouter,           // NEW
  insertedId: string,
  layer: "context" | "shared",
  content: string,
  insertedTags: string[],
  log: RequestLogger,
): Promise<void> {
  // ... existing rag.search + relates-loop + evolveNeighbour ...

  // M-05.2: contradiction detection (best-effort, default off).
  if (!detectEnabled() || drawnNeighbours.length === 0) return;
  try {
    const candidates = drawnNeighbours
      .map(n => {
        const row = n.layer === "context"
          ? memory.getContext(n.id)
          : memory.getShared(n.id);
        if (!row) return null;
        return { id: n.id, layer: n.layer, content: row.content };
      })
      .filter((c): c is ContradictionCandidate => c !== null);
    if (candidates.length === 0) return;

    const verdicts = await detectContradictions(router, log, content, candidates);
    const threshold = minConf();
    for (const v of verdicts) {
      if (v.confidence < threshold) continue;
      const cand = candidates.find(c => c.id === v.id);
      if (!cand) continue;
      try {
        memory.linkEdge(insertedId, layer, cand.id, cand.layer, "contradicts", v.confidence);
      } catch (err) {
        log.warn("post.extractors", `contradiction edge insert failed: ${...}`);
      }
    }
  } catch (err) {
    // Whole detection step is best-effort — never abort linkRelated.
    log.warn("post.extractors", `detectContradictions failed for ${insertedId}: ${...}`);
  }
}
```

`drawnNeighbours` собирается **внутри relates-loop** (push в array только когда `linkEdge` успешен) — не отдельный второй RAG search.

### Caller updates

`extractors.ts` — обе `linkRelated(...)` callsite добавляют `router` в позицию 3 (после `rag`). `router` уже available в `WriteSharedDeps` / `WriteContextDeps` (или ближайшем context bag — subagent смотрит по факту, может быть pass-through через deps). Trivial (≤6 lines).

### Env knobs

- `LINK_CONTRADICT_ENABLED` (default `false` — opt-in! cost-control).
- `LINK_CONTRADICT_MIN_CONF` (default `0.7`).
- `CONTRADICT_MODEL_ROLE` (default `memory`).

### Edge cases handled

- **drawnNeighbours empty** (RAG returned 0 / все self-skipped) → early return, NO LLM call.
- **detectEnabled false** → early return, NO LLM call (default path в prod).
- **LLM throws / returns malformed JSON** → log.warn + return `[]`, no edges drawn, write succeeds.
- **verdict.id не среди candidates** (LLM hallucinates id) → skip silently.
- **confidence < threshold** → skip.
- **`linkEdge` для contradicts throws** (e.g. duplicate edge) → log.warn, не abort loop.
- **`getContext/getShared` returns null** (deleted mid-flight) → filter out before LLM call.

## Тесты

`tests/post-link-related-contradictions.test.ts` (new). Mock router возвращает заранее построенные JSON-strings. Real DB (test-mem5.2-contradict.db).

1. **disabled by default** — `LINK_CONTRADICT_ENABLED` unset → no router.chat call (mock counts invocations), no `contradicts` edges.
2. **enabled, no contradiction** — LLM returns `{"contradicts":[]}` → no `contradicts` edges; relates edges still drawn.
3. **enabled, single contradiction above threshold** — LLM returns `{"contradicts":[{"id":"<seedId>","confidence":0.9}]}` → exactly one `contradicts` edge with weight=0.9 from inserted to seed.
4. **confidence below threshold filtered** — LLM returns confidence=0.3, threshold=0.7 → no edge.
5. **malformed JSON → no throw, no edges** — mock returns "not json blah" → write OK, no `contradicts` edges, log.warn captured.
6. **hallucinated id ignored** — LLM returns id that wasn't in candidates → no edge drawn, no throw.
7. **`drawnNeighbours` empty (no relates) → no LLM call** — no seeded shared/context with matching embedding → mock counts 0 chat invocations.

## Out of scope

- Auto status flip ('active' → 'superseded' / 'stale') при contradiction — отдельный M-08-era / supersedes-flip task.
- Bidirectional contradicts edge (мы рисуем только src=new → dst=existing; reverse direction — это семантика "old contradicts new", тривиально читается через `getEdgesToDst(insertedId, layer, ['contradicts'])` без отдельного INSERT).
- UI surface для conflict review — M-06.5 candidate.
- Multi-step LLM (chain-of-thought / explanation) — single-pass JSON only.
- Cross-layer contradictions weighting — same LLM logic, layer-pair stored as-is.
- A/B benchmark на real corpus — manual eval task.
- `kind='contradicts'` для backfill existing memos — only on new inserts (forward-only).
